# Creator 主文档目录

项目主要文档集中存放于此。各子模块（如 `src/api`、`frontend`）的说明仍保留在各自目录下的 README。

## 文档撰写原则

后续文档按以下原则保持统一风格，便于**展示项目自身结构与进度**，同时**便于参赛与对外审核**：

- **结构可见**：主路径/支线、模块职责、数据流与分工用表格或短列表写清，避免大段 prose 堆砌。
- **进度可勾选**：阶段规划、提交清单、待办项尽量用可勾选列表（`- [ ]` / `✅`），便于自检与评审对照。
- **分工与边界明确**：涉及多系统（如 UniMem、EverMemOS、mesh）时，用「谁负责什么、典型用途」表格或一句话概括，避免职责模糊。
- **入口与链接集中**：关键入口（API、配置、脚本）和跳转链接放在显眼位置；README 与 docs 互相引用，索引一致。

参见 [EVERMEMOS_INTEGRATION.md](EVERMEMOS_INTEGRATION.md)（分工说明）、[ROADMAP.md](ROADMAP.md)（进度与提交待办）、[VIDEO_DEMO.md](VIDEO_DEMO.md)（提交清单与自检）为示例。
