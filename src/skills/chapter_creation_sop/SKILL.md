# 章节创作规范

本规范与 `task/novel` 的默认配置一致，用于写手在生成单章正文时按此 SOP 执行。

## 1. 字数与截断

- **目标字数**：2048 字（推荐落点）
- **允许范围**：1500～3000 字
- **超出处理**：超过 3000 字时做智能截断，保留核心情节与收尾，不硬砍 mid-sentence。

## 2. 情节节奏（占比）

每章按以下结构分配篇幅：

| 段落     | 占比   | 要求 |
|----------|--------|------|
| 开头     | 约 25% | 快速进入情节，抛出本章核心冲突或悬念 |
| 发展     | 约 40% | 展开情节、推进冲突、刻画人物 |
| 高潮     | 约 25% | 冲突到顶点或出现明确转折 |
| 结尾     | 约 10% | 自然收束，为下一章埋钩子，避免烂尾 |

## 3. 对话质量

- **占比**：全章对话字数占正文 20%～40%，避免纯叙述或纯对话。
- **功能**：对话需至少承担其一——推进情节、展现人物、制造冲突、传递关键信息。
- **风格**：角色用语、口吻前后一致，符合人设。
- **技巧**：少用「信息转储」式大段说明，多用潜台词与动作、神态配合。

## 4. 一致性要点

写作时需自检：

- **角色**：性格、立场、说话方式与已有设定一致。
- **世界观**：设定、规则、物理/社会逻辑不前后矛盾。
- **时间线**：事件顺序、时间跨度清晰合理。
- **情节逻辑**：因果合理，不出现无法解释的跳跃或漏洞。

## 5. 与系统的关系

- 本 Skill 为「规范层」说明，具体 prompt 与 token 限制以 `config/novel` 及 `task/novel` 实现为准。
- 编排层或创作助手可在「写本章 / 续写」等场景下，按需将本 Skill 的上下文（level=2）注入 system 或 user，以约束生成质量。
